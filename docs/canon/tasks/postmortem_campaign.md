# Task: Campaign Postmortem

## Purpose
This task analyzes a completed or failed campaign to extract lessons and prevent recurrence of issues.

## Scope
Applies to one completed or failed campaign.

## Required Context
- Active Role: Founder Mode or Marketplace Gatekeeper
- Constitution must be loaded

## Required Inputs
- Campaign timeline
- User communications
- Escrow state transitions
- Outcome (success or failure)

## Postmortem Procedure
The analysis must determine:
- What happened
- What was expected
- Where divergence occurred
- Whether trust debt was created

## Required Output
The postmortem must include:
- Summary of events
- Identified breakdowns
- Trust impact assessment
- Preventive changes required

## Completion Rule
This task is complete only when corrective actions are identified.
