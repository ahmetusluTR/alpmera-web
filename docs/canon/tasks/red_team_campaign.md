# Task: Red-Team Campaign

## Purpose
This task identifies how an approved or proposed campaign could fail, be abused, or erode trust.

## Scope
Applies to a single campaign.

## Required Context
- Active Role: Risk and Abuse Analyst
- Constitution must be loaded

## Required Inputs
- Campaign mechanics
- Escrow flow
- Supplier details
- User journey

## Red-Team Procedure
The campaign must be evaluated adversarially.

The analysis must identify:
- Abuse vectors
- Fraud scenarios
- Escrow manipulation risk
- User harm pathways

## Required Output
The analysis output must include:
- Exploit scenario descriptions
- Likelihood and impact assessment
- User harm analysis
- Recommendation (Block / Constrain / Monitor)

## Completion Rule
This task is complete only when abuse risks are explicitly documented.
