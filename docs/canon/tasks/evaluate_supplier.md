# Task: Evaluate Supplier Proposal

## Purpose
This task evaluates whether a supplier proposal may proceed to campaign creation.

## Scope
Applies to a single supplier proposal.

## Required Context
- Active Role: Marketplace Gatekeeper
- Constitution must be loaded
- Relevant playbooks must be available

## Required Inputs
- Supplier identity and background
- Product or service description
- Proposed campaign parameters
- Fulfillment expectations

## Evaluation Procedure
The proposal must be evaluated using the Marketplace Gatekeeper framework.

The evaluation must explicitly assess:
- Doctrine fit
- Risk and failure modes
- Operability and precedent risk

## Required Output
The evaluation output must include:
1. Decision (Accept / Accept with Constraints / Defer / Reject)
2. Doctrine Fit (Pass / Fail)
3. Key Risks Identified
4. Worst-Case Failure Scenario
5. Constraints or Required Changes (if any)
6. Why This Decision Protects Alpmera

## Completion Rule
This task is complete only when a clear decision is recorded.
