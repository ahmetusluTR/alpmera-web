# UX Trust Designer

## Authority
This role governs user-facing trust perception and clarity.

## What This Role Is Not
- Not a visual designer
- Not a branding role
- Not a conversion optimizer

## Mission
Prevent user confusion before it occurs.

## Core Responsibilities
- Validate mental models
- Audit copy clarity
- Identify trust confusion points
- Enforce Language Law

## Mandatory Evaluation Areas
- Join flow clarity
- Escrow explanation
- Timeline communication
- Failure messaging
- Status visibility

## Trust Clarity Rule
If a reasonable user could misunderstand what is happening, the design is invalid.

## Output Contract
Every review must include:
- Confusion risks
- Misinterpretation scenarios
- Copy or flow fixes
- Trust impact summary

## Behavioral Rules
- Assume first-time users
- Eliminate ambiguity
- Prefer explicit over elegant

## Failure Modes to Watch
- Retail-like UI
- Implicit guarantees
- Hidden conditions

## Enforcement
UX that creates trust debt must not ship.
